Sorting Entities

Sort entities into different holding areas or work lanes based on entity type.

 

The model used for this example, as well as all of the modeling "how-to" examples, can be found in the ProModel Solutions Café (http://www.promodel.com/solutionscafe/howto/).

Solution

  1. Define a storage or activity for accommodating each entity type once it is sorted.
  2. From the activity where the entities are mixed, define a conditional routing (using the entity name as the condition) to each storage/activity.
Example

Three different types of orders (Order A, Order B and Order C) are processed at a Packaging station where they are sorted and sent to one of three different shipping areas (Shipping A, Shipping B and Shipping C) depending on the type of order.

Solution

Each routing from the Packaging activity to each of the shipping areas is a conditional routing with the entity type specified as the condition.
