Percentage Routings

Route to one of several different activities randomly based on a percentage of the output being routed.

 

The model used for this example, as well as all of the modeling "how-to" examples, can be found in the ProModel Solutions Café (http://www.promodel.com/solutionscafe/howto/). A video demonstrating this solution can be found here.

Solution

  1. Connect the FROM activity to two or more TO activities that represent the percentage routings. (A connection may also be made to a blank space on the layout to represent exiting the system.)
  2. In the Routing properties dialog for each of the routing connections, select Percentage as the Route Type (which is already the default) and enter the Percent value which defines the percentage of time the TO activity is to be selected.
Example

Customer orders for nutritional supplements are received by an Order Entry person and then routed to one of three different supply areas (A, B and C) areas for order fulfillment depending on the type (A, B or C) of order. 20% of the orders are of type A, 30% type B, and 50 % type C.

Solution

Notice that only a single entity type (Order) needed to be defined for this model since the three different types of orders are modeling by their percentage of occurrence rather than by testing the actual entity type itself as in a conditional routing (see Conditional Routings). Of course, a new entity name (A, B or C) could have been assigned in each of the routings defined if separate statistics are to be collected based on entity type.
