Attributes

Attributes are place holders assigned to an entity and contain numerical information about that entity. An entity attribute is identified by its name and may be assigned a numerical value for its value. An entity attribute may be examined and acted upon in logic.

Attributes have the following options:

  • Name The name of the attribute.

Variable names must be alpha-numeric, or may contain underscores. No other special characters are allowed such as %, $, !, etc.

  • Type The type of attribute, real or integer.
  • Initial Value The initial value of the attribute to be assigned at the start of the simulation. By default, initial values are 0, but can be changed to whatever value you want.

Attributes vs. Local Variables

Attributes are primarily useful where the value of the attribute is assigned in one logic section and evaluated in another logic section or field, perhaps at a different activity. If, however, an attribute is assigned a value and evaluated within the same logic section, it would be more appropriate to use a local variable. Local Variables act like temporary attributes and are valid only within the logic in which they are defined.
