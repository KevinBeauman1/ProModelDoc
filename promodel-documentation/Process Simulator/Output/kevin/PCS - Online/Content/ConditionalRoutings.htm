Conditional Routings

Route to one of several different activities depending on the type of entity or the value of a variable or entity attribute.

 

The model used for this example, as well as all of the modeling "how-to" examples, can be found in the ProModel Solutions Café (http://www.promodel.com/solutionscafe/howto/). A video demonstrating this solution can be found here.

 

Solution

  1. Connect the FROM activity to two or more TO activities that represent the conditional routing options. (A connection may also be made to a blank space on the layout to represent exiting the system.)
  2. In the Routing properties dialog for each of the routing connections, select Conditional as the Route Type and specify the condition (entity type, attribute or variable) and test value for making the routing decision.
Example

Customer orders for nutritional supplements are received by an Order Entry person and then routed to one of three different supply areas (A, B and C) areas for order fulfillment depending on the type (A, B or C) of order.

Solution

Multiple entity types enter the Receive Orders activity, but they can be correctly routed based on the entity type itself using the conditional routing leaving that activity.
