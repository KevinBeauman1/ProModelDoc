Resource Groups

Resource groups allow you to pool several different resources into one group for use wherever resources may be required.

When a resource group is used in Process Simulator, it is treated as multiple options for use. Each row that is defined in the resource group can be used.

For example, let’s say you have an assembly line task that can be completed by two different Line Operators or an Inspector. You could create a resource group called Assemblers with one Line Op 1, one Line Op 2, and one Inspector. If you choose to use the Assemblers group at an activity, that activity will attempt to capture one of the 3 resources but not all of them.

When a resource group is used in a Get or Use statement in logic and a quantity is specified in the statement details, it becomes a multiplication factor to the quantities defined for the individual resources in the group.  For example, if you have a resource group that has 1 Doctor and 2 Nurses, and you choose a quantity of 2 for the Get statement, 2 Doctors or 4 Nurses will be captured.
