Linking a Submodel

Link to a Subprocess

A quick way to link an activity to a subprocess is to utilize Visio's subprocess capability. To do this, follow these steps:

  1. Select the activity you wish to link to a subprocess.
  2. From the Visio Insert ribbon select the Link to Existing list button.

  1. If you wish to link to a subprocess on another page of the same Visio file, choose the page name listed under Pages in This Document.
  2. If you wish to link to a subprocess in another Visio file, select Browse to Other Document... and choose the file from the Link to File dialog.
  3. With either step 3 or 4 performed, your activity will now be linked as a submodel to the page or document you chose.

Note: If you haven't created a subprocess for the activity yet, you can also choose the Create New button which will create a new page in your document and add the link to it. You can then define the subprocess on the new page at any time.

Inserting a Hyperlink

Another way to link an activity to a subprocess is to add a hyperlink. To do this, follow these steps:

  1. Select the activity you wish to link to a subprocess.
  2. From the Visio Insert ribbon select the Hyperlink button (or use the hot key Ctrl+K).

  1. If you wish to link to a subprocess in another Visio file, enter the Visio file name in the Address field or click on its Browse button and select the name of the file.
  2. If you wish to link to a subprocess on another page of the same Visio file, enter the page number in the Sub-address field or click on its Browse button and select the page name.
  3. Enter an optional description which will be the name displayed in the menu for selecting the subprocess.

  1. Click OK when finished.

Please note: You can link other files to activities as hyperlinks (plain graphics can be linked to separate Visio documents with no effect on the model).

Now when you move the mouse cursor over the activity it turns into a hyperlink symbol designating that a hyperlink has been defined for the activity. To go to the hyperlinked page or file, Ctrl+left-click on the shape or right-click and select the hyperlink name that now appears in the context menu.

Removing/Disabling a Submodel Link

If you no longer want an activity to use a linked subprocess, and instead use its own defined processing, you may either delete the hyperlink or unselect the Use Submodel check box in the Advanced section of the shape properties window.

You can also temporarily "turn off" using a submodel without removing the hyperlink in case you would like to run your model using the parent activity properties to simplify your model for verification or troubleshooting purposes, for example. Then later turn it back on and start utilizing the submodel again without the need to redefine the hyperlink.
