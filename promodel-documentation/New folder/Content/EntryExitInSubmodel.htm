Entry and Exit Points in a Submodel

In addition to linking an activity to a submodel, connections within the submodel must be defined to specify where entities from the parent model enter the subprocess and where entities exit the submodel to return to the parent model. These connections are described below.

Submodel Entry Point

Entities enter a submodel from the parent model at an activity that has an Arrival connection with the Submodel Input checkbox checked in its Advanced section.

Only one entry point in the subprocess should be defined.

Submodel Exit Points

Entities return to the parent model from any activity in the submodel that has a Route to Exit connection with the Submodel Output property checked in the Advanced section of the shape properties. If the Submodel Output checkbox is unchecked, the entities will simply exit the system at that point and report its statistics.

At least one exit point must be defined but a submodel can have multiple exit points.
