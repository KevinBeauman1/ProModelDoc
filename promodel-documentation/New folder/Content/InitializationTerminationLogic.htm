Initialization and Termination Logic

Initialization Logic

Initialization logic occurs at the very start of simulation and allows you to initialize arrays, variables, and other elements. Other common uses of initialization logic include:

  • Displaying messages
  • Prompting for values
  • Activating subroutines that process logic based on a timer

Termination Logic

Termination logic occurs at the very end of simulation and allows you to summarize data collection or write special statistics to an array. Another common use of termination logic is displaying messages.

Execution of Initialization and Termination Logic

It is important to understand exactly when initialization and termination logic are executed. When you run a simulation, the following things occur in the order listed:

  1. Arrays are initialized based upon the defined import specifications.
  2. Variables are initialized to the values specified in the variables table.
  3. Macros and scenario parameters are set to their user-specified values.
  4. Initialization logic is performed.
  5. Simulation begins. Initial arrivals and downtimes are scheduled and simulation processes begin.
  6. Simulation ends.
  7. Termination logic is performed.
  8. Statistics are compiled.
