GroupQty

Function

Syntax samples

GroupQty({<entity name>})

Order GroupQty(Part1) Part2 To Act1
If GroupQty(Part1) > 5 Then...

Description

Returns the number of entities of a specified type in a batched or attached entity. If no name is specified, it returns the entire group quantity. If it is an attached entity, it will only return the number of temporarily attached entities, not the base entity. For example, if four Castings are temporarily loaded onto a Pallet, the GroupQty() will return the number of Castings (i.e., 4), which does not include the base entity Pallet.

In the case of hybrid nested groups with several mixed and nested levels of batches and attached entities, GroupQty() returns the number of entities in the uppermost level only.

If multiple attaches have occurred (no batches), either of different entities or at multiple activities, GroupQty() returns the combined total of all attached entities.

Valid In

Activity, arrival, and route logic. This function returns an integer.

Components

<entity name>

The optional specific type of entity to search for in the group.

Example

A batch of documents called Case arrives at the Secretary in-box and is processed for some amount of time according to a Normal distribution. If the batch contains three or more documents it is routed to Legal Assistant #1, otherwise it is routed to Legal Assistant #2.

If GroupQty() > 2 Then

aDest = LegalAsssistant1

Else

aDest = LegalAssistant2
