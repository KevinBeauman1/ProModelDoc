Connections

Connections are used to model the flow of entities through your simulation. Below is a brief description of the three available connection types. For more information on setting up and defining connections, see Connecting Shapes.

  • Plain Connector   This connection type will have no effect on your model during simulation, and should only be used to add visual elaboration to your diagram.
  • Routings   A routing connects an activity to another activity and defines the movement of entities from one activity to another.
  • Arrivals   An arrival connects an entity to an activity and defines the rate, schedule, or pattern in which entities are created and arrive at the connecting activity.
