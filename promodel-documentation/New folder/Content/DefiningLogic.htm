Defining Logic

Process Simulator logic controls the flow of entities through your process and defines what happens to them along the way. For information and theory on controlling this flow of entities, see Flow Control. Data entered in shape property fields is, in its basic form, a simple form of logic. More controlled or custom logic is defined in Logic windows.

Through Shape Properties

All the data necessary to model a high-level view of a process can be entered through the shape property fields of the objects in your layout. Right-clicking on an object and selecting Properties from the menu will display shape properties for that object.

From the shape properties fields, you can define such items as process times, resource usage, routing decisions, arrival patterns, and other high-level logic. For more information on the details available in shape properties, please see Defining a Process.

Through Logic Windows

Logic allows you to add more detail to your process by including deeper decision-making, tracking, and control. Intellisense and the Syntax Guide assist you in writing correct custom logic. During simulation, the Trace and Debug features help you validate your logic.

This logic is defined within Logic windows found in shape properties for activities, resources, arrivals, and routes, as well as subroutines and Initialization and Termination logic.
