User Distributions

Occasionally, none of the built-in distributions in Process Simulator can adequately represent a data set. In such cases, the user may define a custom distribution (called a User Distribution) to represent an empirically established data set. The user enters the data that defines a User Distribution.

A user-defined distribution is a table of empirically gathered data. User Distributions in Process Simulator support discrete probability distributions. Discrete distributions are characterized by a finite set of outcomes, together with the probability of observing each outcome. To use a User Distribution, you must enter data into the data table for the User Distribution that you define. User Distributions are accessed from the Model Elements group in the Process Simulator ribbon, within the More Elements drop down menu button.

In the screenshot above, three User Distributions (UserDistribution1, UserDistribution2, UserDistribution3) are defined. The empirically gathered data for UserDistribution2 is displayed in the details grid under the Percent and Value columns. In the above example, there are four possible outcomes for the data set of UserDistribution2: 20% of the time the outcome will be 1, 35% of the time the outcome will be 2, 25% of the time the outcome will be 3, and 20% of the time the outcome will be 4. Note that the sum of the percentages in the Percent column must add to 100 percent, as is displayed in the Total on the toolbar. The values in the Value column represent (or symbolize) the outcomes that were observed.

Below are descriptions of the fields and components of the User Distributions grid.

Name — the name of the User Distribution. You can double-click on the name of an existing distribution if you want to rename it. A User Distribution may be named any unique, valid name that will be used in the logic or property field to reference the User Distribution.

Percent — the probability (represented as the percentage) of observing each outcome in the data set.

Value — the value that represents the discrete outcome that was determined empirically. The value must be in the form of either an integer or a real number, but it can symbolize some nonnumerical observation (e.g., a specific health condition).

Total — the total of the percentages in the Percent column. When the entire data set is entered for the distribution, the Total in the toolbar must indicate a value of 100%.
