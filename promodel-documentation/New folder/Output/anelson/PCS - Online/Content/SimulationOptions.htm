Simulation Options

The Simulation Options dialog enables the user to define properties for running the simulation, such as, simulation time and warm-up period. The simulation options can be opened in the following ways:

  1. Clicking on the Simulation Options icon from the Process Simulator ribbon.
  2. Right-clicking anywhere on a blank part of the layout and selecting Simulation Options from the context menu.

Run Length Tab

  • Run Length Type This allows you to specify what time measurement the run time is being specified for; Time Only, Weekly Time, or Calendar Date
  • Warmup Period Allows you to specify whether there is to be a warmup period. Click in the box, if yes.
  • Warmup Length   This is a period in which to get the model to a steady state condition. This is typically only needed if the process is being evaluated for its performance in a steady state. All statistics are reset at the end of a warmup period.
  • Run length (Time Only)   This is the duration for running the simulation. Run length can be defined by a set number of hours, days, or weeks. By default this is set to 40 hours. If shifts are defined, it is important to note that the simulation always starts on Monday morning at 12:00 A.M.
  • Run length (Weekly Time)   Run length may also be defined by weekly time, where you may a Week, Day and Time for the Warmup Begin, Simulation Begin, and Simulation End times.
  • Run length (Calendar Date)   Run length may also be defined by calendar date.  Choose exact dates and times for Warmup Begin, Simulation Begin, and Simulation End times.
  • Adjust for Daylight Saving Time This option will have the simulation account for Daylight Saving Time changes when Run Length Type is set to Calendar Date.
  • Replications   This is the number of statistical variations to run for this simulation. If you place the "@" symbol immediately preceding a number in this field (e.g. @5), you can run the specific replication indicated by that number. Note that you can run a specific replication if, and only if, you have run at least that number of replications previously.
  • Clock Precision  This sets the lowest time unit precision the statistics will be collected at.
  • Clock Precision Units  Select the clock units you want to display at run-time. This also affects the lowest time unit precision the statistics will be collected at.

Animation Tab

  • Enable Animation   When checked, animation will be displayed during simulation. When unchecked the animation is disabled, the simulation will run faster and statistics will still be collected. With this option unchecked, animation cannot be turned on with the run-time animation toggle option.
  • Initial Animation Speed   Use the slider to set the speed of the animation when the simulation begins.
  • Unit for Avg CT Display   Allows you to select the unit of time that will be used for the Average Cycle Time statistic displayed at the end of an Exit route during simulation.
  • Pause Simulation   Pauses the simulation before it begins.
  • Show Trace   Opens the Trace window at the start of simulation and puts it in Step mode.
  • Configure Page Layout Options allows you to choose how many diagram page layouts are placed on a row during simulation. You can either limit the number of page layouts per row based on a boundary specified in pixels or by the number of pages.
    • Layout Type
      • Boundary in Pixels This specifies the maximum amount of space (in pixels) allowed before page layouts are added to a new row.
      • Number Per Row This sets the specific number of pages allowed before subsequent pages are added to a new row.
    • Start Submodel Layout on New Row Forces the first submodel to be added to the layout on its own row (regardless if the above limit has been reached). This ensures that only main model pages are organized together.

Click OK to save the property settings or Cancel to exit without saving.
