Xwrite

Statement

Syntax samples

Xwrite <file ID>, <string or numeric expression>

Description

Writes information to a text file in any text format the user chooses. Xwrite is for writing user-formatted files, while Write and Writeline are for writing special formatted text files. Xwrite always appends to a file unless the file is Reset. Note that any time a Write or Writeline writes to a file, the file will automatically be a text file. No end of file marker is appended to files written only with Xwrite. In subsequent replications, additional items are appended to the end of the file unless the file is Reset. Process Simulator does not format the string expression, although you can use the Format function to manually format data. To add an end of file marker to a user-formatted file, use Xwrite Char(26).

Valid In

Any logic.

Components

<file ID>

The name of the file as previously defined in the External Files grid.

<string or numeric expression>

The string expression to be written to the file.

Example

The following example uses Xwrite to record the time each Box completes processing at the activity Shipping.

Wait N(7.3, .4)
Xwrite Rpt, "Box shipped at: " $ Format(Clock(min), 3, 2)

See Also

Write, Writeline, and Reset. Also see External Files on page 1
