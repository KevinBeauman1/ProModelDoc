Log

Statement

Syntax samples

Log <string>, <expression>

Log “Activity Time”, Attr1

Description

Process Simulator assumes that the time stored in the expression is in the model’s default time units, which is minutes. Use the Log statement to record the time from one point in the process to another by storing the time of the initial point in an attribute, variable, or array element with the Clock function and using the same attribute, variable, or array element as the expression in the Log statement. Use the Log statement to determine throughput time or throughput in a section of the facility.

Valid In

Activity, arrival, and routing logic.

Components

<string>

This string is used to identify the logged numbers in the statistics shown in Output Viewer.

<expression>

The result of this expression subtracted from the model’s current simulation time will be reported under the <string> identifier above.

Example

The example below shows a Log statement used to capture each Client’s total throughput time as they travel through the system. Time starts when the Client arrives at Receptionist and stops when the Client exits through the door. The first process sets attribute CT equal to the current clock time. Next, Clients are sent randomly to one of three offices. Finally, when a Client leaves the system at location OutDoor, the Log statement records the cycle time by subtracting the time stored in attribute CT from the current simulation time.

Reception Activity Logic

// Capture the initial time
CT = Clock()

OutDoor Activity Logic

// Log the cycle time
Log "Cycle Time", CT
