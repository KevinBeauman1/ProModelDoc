Attach

Quantity  This allows you to specify the number of entities to be attached to the entity waiting at the connecting activity.

Quantity Mode Allows you to select the behavior by which to attach the specified quantity.

  • Entire Quantity With this option, the base entity will attach the total number of entities defined in the Quantity field and wait until all entities are available to be attached.
  • Immediately (0 to Qty) With this option, the base entity will attach whatever quantity is available at the activity and immediately route out.
  • With Time Limit (0 to Qty) For this option, the base entity will attach the total number of entities defined in the Quantity field within the specified time limit. If the attach quantity has not been met when the time limit is reached, the base entity will continue on with the entities it has attached.

Attach To Defines which entities will be attached to the base entity. Clicking the drop down list will show the following options:

  • Any Entity Allows any entity type to be attached to the base entity.
  • Creator Restricts the attaching entities to only those that the base entity created through a Create routing earlier in the process.
  • Selected Entities Only Choosing this option shows an additional field where you can select specific entities to attach from a drop down list.

When Allows you to specify whether the attach occurs before or after the base entity executes its activity logic.

Condition This is optional and allows you to more specifically choose to attach entities based on their attribute value.

Permanently You may also choose to permanently attach the entities, which means that there is no intention of detaching the entities later in the process. By doing this, all cost statistics of the attaching entities are rolled up and reported with the base entity.
