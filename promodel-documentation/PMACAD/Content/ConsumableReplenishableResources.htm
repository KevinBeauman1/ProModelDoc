Consumable Replenishable Resources

Certain types of resources are consumed or replenished by an activity. Examples include energy, water and money.

 

The model used for this example, as well as all of the modeling "how-to" examples, can be found in the ProModel Solutions Café (http://www.promodel.com/solutionscafe/howto/). A video demonstrating this solution can be found here.

Solution

  1. Define a variable to represent the resource and assign an appropriate initial value representing the number of BTUs, gallons, dollars, etc. that are to be available at the start of the simulation.
  2. At activities where the resource is consumed or replenished, click on the Logic in the Activity Properties and define the appropriate logic.
  1. Decrement or increment the variable to reflect resource usage and replenishment.
  2. Use an assignment statement to reset the level of the resource to a specific level.
  3. Use a WAIT UNTIL statement if it might be necessary to wait until the resource has a particular level value before it can be used.
  1. Statistics on changes in value of the resource can be viewed as output from the simulation.
Example

A Powder Coat paint operation uses a powdered coating process to paint chairs. Each chair uses .6 lbs of paint and when the powder paint reservoir reaches 10 lbs., it is replenished back up to 300 lbs.

Solution

A variable (vPaint_Level) has been defined with an initial value of 300. This variable is decremented by .6 whenever the Powder Coat activity occurs. When the vPaint_Level variable drops to less than 10, it is reset to 300.
