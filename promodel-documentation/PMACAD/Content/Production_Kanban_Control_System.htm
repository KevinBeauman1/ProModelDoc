Production Kanban (Reorder Point) Control System

Produce entities at an upstream activity when a signal is received that a downstream inventory has dropped to a specified inventory level. (Note: this is just like a conveyance kanban only entities begin an activity rather than conveyed or moved.)

 

The model used for this example, as well as all of the modeling "how-to" examples, can be found in the ProModel Solutions Café (http://www.promodel.com/solutionscafe/howto/).

Solution

  1. For the activity where entities are to be produced, define a buffer prior to the activity with a Send routing to the activity.
  2. For the buffer or storage that is to be replenished when its inventory drops to a specified trigger level, define a storage by dragging a storage shape onto the layout.
  3. Fill in the information for the storage.
Example

Whenever a finished goods inventory of Tables drops to 10, an additional 100 tables are assembled and sent to finished goods to replenish the inventory.

Solution

The Buffer has an ordered routing to the Assembly activity. From the Assembly activity finished tables are then routed to Finished Goods inventory. The storage dialog for the Finished Goods Inventory has the appropriate trigger level and reorder quantity specified.
