Scenario Manager

In Process Simulator it is possible to set up multiple scenarios to be run and evaluated side by side in the output report to see how they compare. Scenarios are defined in the Scenario Manager dialog, which is accessible from the Process Simulator ribbon.

The scenario manager provides a table where multiple scenarios can be defined, each with a set of custom parameters. Parameters are values for simulation object properties, for example an activity’s capacity, or a resource’s number of units. These values, or parameters, can be easily changed in the scenario manager and varied for different scenarios. This is helpful for comparing changes to the model’s simulation objects without creating and managing multiple models.

Toolbar

The Scenario Manager toolbar contains the following options:

  • Add Parameters: this button adds rows to the table, which allows you to vary the value of the parameter for each scenario.
  • Add Scenario: this button adds columns to the table, which represent different scenarios.
  • Move Row Up/Down these buttons move the selected rows to different positions in the table.
  • Show Baseline Values checked by default, all values in the table will be shown. Unchecked, it will hide any values that are the same as the baseline value.

The Baseline column is not editable, and shows the values of the parameters as defined in the model.

The Simulate Scenario? row allows you to choose the scenarios you would like to have simulated when you click the Run Scenarios button, or choose Simulate Scenarios from the Process Simulator ribbon.

Last Run Date displays the last date and time the scenario was simulated. If blank, the scenario hasn't been simulated or the results were deleted.

Right-Click Menu

Right-clicking in the Scenario Manager will display a menu with options pertaining to the area you clicked on. Using the right-click menu can speed up the process of adding of scenarios and parameters.

Reset Parameter Name - this will reset a parameter name, that has been modified, back to its original name.

Reset Value To Baseline - this will reset the parameter value, that has been modified for a specific scenario, back to the same value as the baseline.

Disconnect From Baseline - if a parameter value for a specific scenario has not been modified, it is connected to the baseline value and will be displayed unbolded in the field. Choosing this option will keep the value but treat it as if it has been modified (and will be displayed bolded).
