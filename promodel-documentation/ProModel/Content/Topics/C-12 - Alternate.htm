Alternate

Syntax samples

ALT

Description

Causes a location to be selected as an alternate destination if it has available capacity and the condition for a preceding routing rule is not met. If the ALTERNATE location is unavailable, the entity waits until it becomes available or until the preceding routing rule is satisfied.

 

ALTERNATE routings are common to all primary routings, except the Probability and User Condition rules, and should therefore be listed after the last primary routing but before any backup routings.

 

For example, a high-speed machine might be preferred, but alternatively, a slower machine can be used when the faster machine is unavailable.

Valid In

The rule field of the Routing edit table only. An ALTERNATE routing may be specified after any other type of routing (including other ALTERNATE routings) except for Backup, CONTINUE and DEPENDENT routings.

Example

After a Client completes a 3 minute interview with the Receptionist, he or she is routed to either Office1 or Office2 if capacity is available at one of those locations. If no capacity is available, an alternate routing will direct the Client to a waiting area. If no capacity is available in the waiting area, the Client will remain at the reception location until capacity becomes available at any of the three locations. After Patient completes a 20 minute treatment at Loc1, he or she is routed to either Loc2 or Loc3 if a Send statement somewhere in the model has directed a patient to be sent to one of those locations. If no Send request is waiting to be filled, an alternate routing will route the patient to Loc4. If no capacity is available at Loc4, the patient will wait until capacity becomes available at Loc4 or a Send statement that matches the entity and the location in one of the Send routings is executed somewhere in the model. After Shaft completes a 3 minute operation at Drill, it routes to either Lathe1 or Lathe2 if a Send statement somewhere in the model directs an EntA to be sent to one of those locations. If no Send request is waiting, an alternate routing will route the entity to Lathe3. If no capacity is available at Lathe3, the entity will wait until capacity becomes available at Lathe3 or a Send statement that matches the entity and the location in one of the Send routings executes somewhere in the model.
