Pre-translation check for Path Networks and Automatic Mapping

When you run your simulation, MedModelProModelServiceModel will compile your model, check your path networks for errors and automatically calculate shortest distances between nodes. If your model has large, complex path networks, this could cause your model’s compilation time to run quite long.

 

To combat this, MedModelProModelServiceModel will perform a pre-translation check for path networks. This means that the first time your model compiles, it will check your path networks for errors and automatically calculate the shortest distance between nodes. It will then save this information.

 

Every subsequent time your model compiles, it will read the saved information about your path networks and not check for errors or recalculate shortest distances. This saves compilation time on large models.

 

However, if you edit one or more of your path networks, MedModelProModelServiceModel will once again check the modified path networks for errors and recalculate shortest distances during compilation. MedModelProModelServiceModel will consider a path network to have been modified if its name was at any time, since the last compile, highlighted in the Path Network dialog window.

 

Or if you have checked to recompile mappings in the simulation dialog, MedModelProModelServiceModel will check all Path networks for errors and recalculate shortest distances (i.e. automatic mappings) between each scenario.
