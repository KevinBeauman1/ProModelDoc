Syntax samples

BreakBlk

Description

Exits from the innermost statement block. The next statement to be executed will be the one immediately following the End statement of the innermost statement block. If a BreakBlk is executed outside any statement block, MedModelProModelServiceModel will exit the logic completely.

Valid In

Any logic.

Example

This example uses BreakBlk in the logic for a shipping department that handles two types of entities, Letters and Packages. Both packages and letters need to be addressed, but packages may need to be wrapped before they are addressed. When a package arrives, shipping increments the variable No_of_Pkg to keep track of the number of packages it ships. Attr1 is then checked to see if the package has been wrapped. If the package has been wrapped (if Attr1 = 1), the BreakBlk statement exits the statement block that wraps packages. If the package has not been wrapped, a resource, Wrapper, wraps the package. Finally, all entities that Shipping processes are addressed by the same statement and routed to the location Post_Office. The line “// Breakblk to here” is a comment which marks the next statement executed after the BreakBlk.
This example uses BreakBlk in the logic for a Central Material Service (CMS) that handles two types of entities, Sterile Trays (Tray) and Solutions (Solution). Both trays and solutions need to be processed, but trays may need to be wrapped beforehand. When a tray arrives, the CMS increments the variable No_of_Trays to keep track of the number of trays it processes. Then it checks Attr1 to see if the tray has been wrapped. If the tray has been wrapped (if Attr1 = 1), the BreakBlk statement exits the statement block that wraps trays. If the tray has not been wrapped, a technician, (Tech) wraps the tray. Finally, all entities that CMS processes are labeled by the same statement and routed to the holding location. The line “//Breakblk to here” is a comment which marks the next statement executed after the BreakBlk.
