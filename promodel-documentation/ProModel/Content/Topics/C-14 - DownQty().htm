DownQty()

General System Function

Syntax samples

DownQty(<location> or <resource>)

If DownQty(Loc1) > 3 Then Route 2

Display “Total Res1 Down Now:” $ DownQty(Res1)

Description

Returns the number of location or resource units down at the time of the call. Use this function to make decisions based on how many resource or location units are down. For example, if too many units are down, a foreman could preempt several units back into service.

Valid In

Any logic and all fields except those fields evaluated at translation only. For a list of the fields evaluated only at translation, see Appendix A on page 1.

Components

<location>

The name of the location to examine. Loc() can be substituted for the name of a location.

<resource>

The name of the resource to examine. Res() can be substituted for the name of a resource.

Example

Two resources, Workers, wash cars at location Man_Wash in 6.5 minutes. An automatic car wash, Auto_Wash, can also perform the task, but it takes 9.3 minutes. If only one Worker is available (i.e., the other Worker is down), it takes a single Worker 13.0 minutes to wash the car. Therefore, if one or more Workers are down, we want to route cars to the automatic car wash, Auto_Wash. An Acute Minor Illness Clinic (AMIC) located next to the emergency room is staffed by one physician. If the physician has gone to lunch or is on break, minor illness patients are sent to the Emergency room for care. Two resources, Welders, weld brackets onto steel frames at a location, Man_Weld, in 6.5 minutes. An automatic welding machine, Auto_Weld, can perform the same task, but it takes 9.3 minutes. However, if only one Welder is available and the other Welder is down (e.g., on break), it takes one Welder 13.0 minutes to weld the brackets to the frames. Therefore, if one Welder is down, the frames should route to the automatic welding machine, Auto_Weld.
