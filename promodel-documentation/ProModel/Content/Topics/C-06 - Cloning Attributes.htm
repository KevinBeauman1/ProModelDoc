Cloning Attributes

Whenever one entity initiates the creation of other entities through a Split As, Create, or Order statement, or as the result of specifying multiple outputs in a routing, the attributes of the original entity are automatically copied to each newly created entity.

 

The following examples show two methods of splitting an incoming group of entities, called Shipment, into six Items. The first method uses a Create statement in the operation logic to create the new entities called Item. The original entity, Shipment, will have its name changed to Invoice in the routing table. In the second example, both of the new entity types are created in the routing table, so no Create statement is needed in the operation logic. In both cases, the attribute attached to the original entity, Quantity, is duplicated in both Invoice’s and Item’s attributes.

 

The following examples show two methods of initiating the creation of entities which bear the same attribute values as the parent entity. The first method uses a Create statement in the operation logic to create an entity called “Blood_samp.” The second example uses multiple output routings to send the patient to the waiting room and his or her medical record (Med_rcrd) to the desk. In both cases, the attributes attached to the original entity, Patient, are duplicated in both the Blood_samp and Med_rcrd attributes. As such, both will carry with them the original patient’s identity value (Patient_ID).

 

The following examples show two methods of splitting an incoming batch of material, called BatchA, into one Tote and six EntA’s. The first method uses a Create statement in the operation logic to create the new entities, called EntA. The original entity, BatchA, will have its name changed to Tote in the routing table. In the second example, both of the new entity types are created in the routing table, so no Create statement is needed in the operation logic. In both cases, the attributes attached to the original entity, BatchA, are duplicated in both Tote’s and the EntA’s attributes.
