ResQty()

Entity-Specific System Function

Syntax samples

ResQty({<resource name>})

If ResQty(Res1) > 5 Then Free 5 Res1

Description

Returns the number of units of the specified resource that the current entity owns. ResQty() can be used to determine the amount of time necessary to process an entity based on the number of resources an entity owns.

Valid In

Entity speed fields, traveling-time fields, resource fields, operation logic, routing fields, arrival logic, debug user-condition fields, and move logic. This function returns an integer.

Components

<resource name>

The name of the resource to check for. If no resource name is specified, the total number of units of all resource types owned by the entity is returned. Res() may be substituted for a resource name.

Example

The example below demonstrates the use of ResQty() to adjust the processing time for an entity based on the number of resources available to process it. Once a Plane arrives at a passenger gate, it captures a certain number of resources named BH (Baggage Handlers). Operation logic at the gate determines how many BH resources have been captured and, accordingly, how long it will take to service the Plane. The more Baggage Handlers a Plane has captured, the less time it takes to service it. Either an RN or an LPN may perform an initial examination of a patient. A nurse will take N(30,5,1) minutes to perform the exam while an LPN will take N(40,5,1) minutes. In the example below, either resource is requested and the ResQty() statement is used to determine which one is obtained by testing for the presence of the nurse and then executing the appropriate logic. The example below demonstrates the use of ResQty() to adjust the processing time for an entity based on the number of resources available to process it. Before EntA’s arrive at Loc1, they capture a certain number of resources named Res1. Processing logic at Loc1 determines how many Res1’s each EntA captured and processes it accordingly. The more resources an entity captures, the more workers available to work on the project, and the less time it takes. ProModel then routes EntA’s on to Loc2. The logic at Loc2 makes sure that no EntA owns more than one Res1.
