Autodesk®Edition Home Tab

The Autodesk® Home ribbon tab provides access to most of the modules for developing your model, simulating and creating scenarios, and various other common functions of the program. As you hover your mouse over the buttons in the ribbon, tooltips will appear describing the module as well as showing hotkeys for quick keyboard access to the module.

Exclusively in ProModel AutoDesk® Edition, the Home ribbon has added the additional functions for Autodesk® users.

The selections from the Autodesk® Edition Home Tab are as follows:

Layout Only Closes the current module and places the model in Layout Only mode, which allows all element types to be multi-selected and repositioned as desired.

Locations Locations are places in the model where activities happen. See Locations on page 1

Entities Entities are objects that have actions performed on them. See Entities on page 1

Arrivals Define which entities enter the model during simulation. You can specify in what quantity, at which locations, and the frequency in which they arrive. See Arrivals on page 1

Processing Defines the flow of entities through the model. Define what activities and tasks happen, where they occur, where entities route to next, and how they get there. See Processing on page 1

Resources Resources are static or mobile agents that perform actions, usually on entities. They represent people, machines or tools that are limited in quantity and thus constrain the system. See Static Resources on page 1 or Dynamic Resources on page 1

Path Networks Define the course of travel for dynamic Resources. They represent hallways, aisles, tracks, roadways, etc. where people or vehicles move. See Path Networks on page 1

Background Graphics Add text, images, a floor layout, or other shapes as background graphics to enhance the visual representation of your actual system. See Background Graphics on page 1

Simulate Initiates a simulation run based upon the settings in the Simulation Options dialog.

Simulation Options Provides you with several options to control the simulation, such as run length, number of replications, disabling animation, etc. See Simulation Options on page 1

Simulate Scenarios Initiates a run that simulates all scenarios selected in the Scenario Manager.

Scenario Manager Define multiple scenarios, vary their common parameters, and choose which ones to simulate for side by side analysis in Output Viewer. See Scenario Manager on page 1

Attributes Attributes are used for uniquely tagging entities with important data. They are useful for identifying entities during processing, executing conditional logic, or for tracking custom statistics. See Attributes on page 1

Variables Variables are place holders you define to store changing or static numeric values. They are used in calculations and other model logic, and can have a counter graphic to dynamically display values during simulation. See Variables on page 1

Arrays An array is a matrix of cells that contain values like in a spreadsheet. Arrays can be initialized by data in Excel files or a database and can export their data at the end of simulation. See Arrays on page 1

Macros Macros are place holders for often used expressions in your model. They can contain numbers, a set of statements and functions, or parameters that initialize model elements such as resource units, routing probability, etc. for varying from scenario to scenario. See Macros on page 1

Subroutines A user-defined command that can be called to execute a block of logic or perform a calculation and optionally return a value. Define parameters to pass as arguments with values unique to the logic or field from which they are called. See Subroutines on page 1

Shift Assignments Assign schedules to resources and locations, which specify daily work hours and time off such as vacation. See Shift Assignments on page 1

Arrival Cycles An arrival cycle is a pattern of individual arrivals that occur and usually vary over a given time period. After a cycle is defined, you use it by adding it to the Qty Each field for any Arrival. See Arrival Cycles on page 1

External Files Specify files that contain external data used during simulation for things such as operation times, arrival schedules, process flow, etc. Also, define files to write out or log data to during simulation, for example, for model validation or custom statistics. See External Files on page 1

Find Find text or other content in the model.

Replace Search for text you'd like to change, and replace it with something else.

Find Next Continue to the next instance of the text being searched for.

Add Assets To Library Allows user to Add Autodesk Assets to the Model's Graphic Library.

Workflows Allows user to Open the assets used to build the simulation model with AutoCAD® or Inventor®.
