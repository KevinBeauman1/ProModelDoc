ForLocation()

Shift and Break System Function

Syntax samples

ForLocation()

If ForLocation() Then Priority 100

If ForLocation() Then

Inc Arr1 [1, 2]

Else

Inc Arr1 [2, 2]

Description

This function returns TRUE if the object executing the shift or break logic is a location.

Valid In

Shift or break logic.

Example

Three resources (Agent1, Agent2, and Agent3) and two locations (Counter1 and Counter2) are assigned to a shift. When the locations go off-shift a resource called Manager is used for 10 minutes to close the counters. Because the same shift is assigned to locations and resources, we need to check if the object executing the off-shift logic is a location. The following logic is placed in the off-shift logic: Three resources (Tech1, Tech2, and Tech3) and two locations (Lab1 and Lab2) are assigned to a shift. When the locations go off-shift a resource called Housekeeping is used for 10 minutes to clean the areas. Because the same shift is assigned to locations and resources, we need to check if the object executing the off-shift logic is a location. The following logic is placed in the off-shift logic: We assign three resources (Oper1, Oper2, and Oper3) and two locations (Mill1 and Mill2) to a shift. When the locations go off-shift, the example uses a resource called Maint for 10 minutes to clean up around the machines. Because we assigned the same shift to locations and resources, we need to check if the object executing the off-shift logic is a location. We place the following logic in the off-shift logic:

If ForLocation() Then Use Manager For 10 min If ForLocation() Then Use Housekeeping For 10 min If ForLocation() Then Use Maint For 10 min
