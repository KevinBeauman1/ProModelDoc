Memory Allocation for Attributes

Because locations always exist during a simulation, location attributes always use memory. However, attributes for entities are not created until the first time any of the entity’s attributes are examined or set. At that time, MedModelProModelServiceModel allocates enough memory for all of the entity’s attributes. When the entity exits the system or is absorbed by another entity through a Combine or Join statement, MedModelProModelServiceModel automatically frees the memory allocated for its attributes.
