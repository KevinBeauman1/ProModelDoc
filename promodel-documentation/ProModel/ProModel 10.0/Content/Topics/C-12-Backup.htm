Backup

Description

Locations specified by this rule are selected only if all of the destinations (primary or alternate) listed in the preceding routings of the same block are unavailable due to any downtime, including a shift. One use of Backup is to handle a machine that is not functioning, and entities need another destination to continue.

Valid In

Inside the rule field of the Routing edit table, at the end of a routing block, after one or more primary routings or even an ALTERNATE rule. Conditions for using Backup routings with either USER CONDITION and PROBABILITY routing rules are given in the notes for these rules.

Example

After a Claim completes a three minute operation at Station1, it is routed to Station2 when Station2 becomes available. Only if Station2 and the alternate location (Station2A) are both down will it be routed to Station3. If Station3 is also unavailable or down, the Claim will wait for Station3 only as long as both Station2 and Station2A remain down. After an entity, Patient, completes a 30 minute treatment at Loc1, he or she is routed to Loc2 when Loc2 becomes available. Only if Loc2 and the alternate location (Loc2A) are both down will the patient be routed to Loc3. If Loc3 is also unavailable, the Patient will wait for Loc3 only as long as both Loc2 and Loc2A remain occupied. After an entity, Chip, completes a 3 minute operation at CNC, it is routed to Stamp1 when Stamp1 becomes available. Only if Stamp1 and the alternate location (Stamp2) are both down will it be routed to Stamp3. If Stamp3 is also unavailable, Chip will wait for Stamp3 only as long as both Stamp1 and Stamp2 remain down.
