Defining Entities

Once all locations have been defined, we define entities in a similar way by selecting an icon for each entity type. As we do this, a record is created in the Entity edit table for each entity type. For more information on how to define entities, see Entities on page 1.

In this model the Speed (fpm) column is irrelevant since all entities move on a conveyor or according to the definition of a mobile resource. Also, the Stats column shows that we desire detailed statistics for all entity types. “Time series” statistics include throughput history of the entity. “Basic” statistics include only the total exits of each entity type from the system and the final quantity of each entity type in the system.

Defining Entities

Once all locations have been defined, we define entities in a similar way by selecting an icon for each entity type. As we do this, a record is created in the Entity edit table for each entity type. For more information on how to define entities, see Entities .

In this model the Speed (fpm) column contains the default value of 114 fpm. This is the rate of movement the patient will use if traveling on a path network. Also, the Stats column shows that we desire time series statistics for all entity types. “Time series” statistics include time series data such as content and duration histories for each entity type at each location. “Basic” statistics include only the total number of each entity type of patient that leaves (exits) the system and the final quantity of each type of patient remaining in the system.
