Defining Process Logic

The last step in executing Phase 1 of your model building is to define the processing of entities at each location. ProModel simplifies this task by allowing you to select an entity type and then use the mouse to click on the locations in the order in which they will process the entity. Each time you click on a location, a new processing record is added to the Process edit table, defining the process for that entity type at that location. For more information on defining process logic, see Processing, and Operation Logic.

Process editing actually involves two edit tables that normally appear side by side. The Process edit table specifies what happens to an entity when it arrives at a location, and the Routing edit table specifies where an entity is to be sent once processing is complete.

Operation and Move Logic

Once the basic entity flow has been defined using the point and click method, logic statements are added to Operation logic. The logic can be as simple as a constant operation time or as complex as a nested If...Then...Else statement.
