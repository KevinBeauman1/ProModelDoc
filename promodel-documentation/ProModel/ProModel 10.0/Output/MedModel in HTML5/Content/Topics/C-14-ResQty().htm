ResQty()

Entity-Specific System Function

Syntax samples

ResQty({<resource name>})

If ResQty(Res1) > 5 Then Free 5 Res1

Description

Returns the number of units of the specified resource that the current entity owns. ResQty() can be used to determine the amount of time necessary to process an entity based on the number of resources an entity owns.

Valid In

Entity speed fields, traveling-time fields, resource fields, operation logic, routing fields, arrival logic, debug user-condition fields, and move logic. This function returns an integer.

Components

<resource name>

The name of the resource to check for. If no resource name is specified, the total number of units of all resource types owned by the entity is returned. Res() may be substituted for a resource name.

Example

Either an RN or an LPN may perform an initial examination of a patient. A nurse will take N(30,5,1) minutes to perform the exam while an LPN will take N(40,5,1) minutes. In the example below, either resource is requested and the ResQty() statement is used to determine which one is obtained by testing for the presence of the nurse and then executing the appropriate logic.
