Do...Until

General Control Statement

Syntax samples

Do <statement block> Until <Boolean expression>

Do Inc Var1 Until Array1[Var1] <> 10

Do

{

Inc Var2, 5

Wait 5 sec

}

Until FreeCap(Loc1) > 5

Description

Repeats a statement or statement block continuously while a condition remains false. Do...Until is an exit-condition loop, meaning that the loop will always be executed at least once. Use Do...Until when an operation will always be executed at least one time and possibly more times.

Valid In

Any logic.

Components

<statement block>

The statement or block of statements to execute.

<Boolean expression>

As long as this expression is FALSE, the loop will continue. This expression is evaluated for each iteration of the loop.

Example

The original (File_cpy) of a multi-part form is sent to a location called “file” where it waits until all other copies of the form are returned following processing. As each page of the form returns it is attached to the file copy. When all copies have returned and are attached to the original, the file is sent to the medical records office for final processing. The logic in the following example models this situation with a Do...Until loop. Because all copies of the multi-part form are not used each time, Attr1 is used to record how many copies were used and therefore must return before the packet is complete. Also note that Attr5 is used to make certain that each copy is correctly matched with the original copy before it is joined to it.
