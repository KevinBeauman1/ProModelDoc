Defining Downtimes

Downtimes may be defined in either the Shift Assignments, Locations or Resources module. For this example, the Resource module is employed because the break is unscheduled and no daily shift schedule has been established to represent multiple work days.

In the example above, we have defined a recurring (i.e. every day at the same time) 30 minute interruption, where the frequency is set to 24 Hr indicating the downtime will occur every 24 hours. First Time is set to 4 Hr indicating the interruption will occur four hours into the simulation run, and Logic is set to 30 indicating that the interruption will last 30 minutes. Also note that the doctor will go to node 13 and remain there for the full duration of the downtime.
