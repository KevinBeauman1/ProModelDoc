Cloning Attributes

Whenever one entity initiates the creation of other entities through a Split As, Create, or Order statement, or as the result of specifying multiple outputs in a routing, the attributes of the original entity are automatically copied to each newly created entity.

The following examples show two methods of initiating the creation of entities which bear the same attribute values as the parent entity. The first method uses a Create statement in the operation logic to create an entity called “Blood_samp.” The second example uses multiple output routings to send the patient to the waiting room and his or her medical record (Med_rcrd) to the desk. In both cases, the attributes attached to the original entity, Patient, are duplicated in both the Blood_samp and Med_rcrd attributes. As such, both will carry with them the original patient’s identity value (Patient_ID).
