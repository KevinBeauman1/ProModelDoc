Object Explorer

The Object Explorer displays a tree view of all the objects in your model. This is helpful for understanding how the shapes and connections in your model interact. The Object Explorer is also useful for reordering.  This is done by selecting one of the items in the list and dragging it to the new position. The connections between activities may be moved to determine the order that entities will route from one activity to several available activities.

For example, if you have entities routing from multiple activities to an activity using the Attach (see Attach on page 1) routing type, the order in which the destination activities are chosen for routing is determined by the order of the listed attach routings in the Object Explorer.

Object Explorer Toolbar

  • Object Filter  This drop down list gives options to view the simulation objects for the Main Model (all pages marked as type Main) or each individual page within your document. If a page is marked as type Submodel or Plain, this will be designated within parentheses next to the page name in the list.
  • Refresh  The Refresh button allows you to reload the list of objects, which you will need to do if you make changes to your diagram and want to see those changes reflected in the Object Explorer. In other words, the Object Explorer does not automatically update whenever changes are made (e.g. adding, renaming or removing an activity). You must manually update the Object Explorer by clicking the Refresh button.
  • Collapse All  This button will collapse all expanded folders down to the root elements (as can be seen above).
  • View Object  With this button toggled on, as you select a shape in Object Explorer, the layout in Visio will be panned to bring that shape into view. If the shape is on a page other than the active one, its page will be activated and the shape will be brought into view.
  • Show Index Number  With this button toggled on, as you select a shape in Object Explorer, the layout in Visio will be panned to bring that shape into view. If the shape is on a page other than the active one, its page will be activated and the shape will be brought into view.

Note: the ability to show or hide index numbers is a feature available only in the Professional edition of Process Simulator. Index numbers are needed in referencing activities, entities, and resources in array cell definition or in statements through system functions like Loc(), Ent() and Res().
