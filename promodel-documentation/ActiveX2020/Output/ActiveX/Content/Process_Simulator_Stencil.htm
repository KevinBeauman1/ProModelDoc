Process Simulator Stencil

The Process Simulator stencil is the default stencil opened when you choose a simulation drawing type, or start Process Simulator from its Desktop shortcut icon. Although you may build simulation models using shapes from any stencil, you must either start with the Process Simulator template or convert an existing Visio document in order to use Process Simulator.

 

The Process Simulator stencil has many shapes you will find useful to model your processes. These shapes have predefined simulation types, which are defined by the naming conventions used in the names of each shape on the stencil. You may edit the stencil to add or remove shapes specific to your organization. In addition to the Process Simulator stencil, several other stencils are installed with Process Simulator:

  • Manufacturing
  • Healthcare
  • Services
  • Value Stream Map

Naming Conventions

Shapes in the Process Simulator stencil have names, which determine what type of object the shape will become when placed on the layout. The default shapes in the Process Simulator stencil have a descriptive name and a type name.

  • The descriptive name is the first part of the name of the shape.
  • The type name, which is contained in parentheses or brackets, determines the simulation object type (activity, buffer, storage, entity, resource, or plain graphic) of the shape.
    • Type names contained in parentheses, ( ), will cause the shape to assume the descriptive name when it is placed in the layout. For example, a stencil shape with the name "Worker (Resource)" will be given the name of "Worker" (with a number appended at the end) and be a Resource simulation object when placed on the layout.
    • Type names contained in brackets, [ ], will cause the shape to assume the type name when it is placed in the layout. For example, a stencil shape with the name "Event [Activity]" will be given the name of "Activity" (with a number appended at the end) and be an Activity simulation object when placed on the layout. If an object called Activity already exists in the layout, the new object will be called Activity 2, and so on.
  • Shapes without a type name will automatically be assigned as an Activity type when placed on the layout.

Applying these naming conventions to new or existing shapes from a stencil helps to speed up model building by automatically applying the correct type to each object placed on the layout.
