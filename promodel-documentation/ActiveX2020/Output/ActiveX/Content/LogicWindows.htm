Logic Windows

The Logic windows provide a convenient interface for entering custom logic for the objects in your process. If you're not sure where to start with logic, the Statement List on the toolbar, Intellisense, and Syntax Guide are features that will help you.

Logic Window

  • The Logic window displays the logic you have typed in it and the order in which it will be executed. Logic is executed from top to bottom each time this particular logic is called during simulation.
  • Type the logic directly into the window immediately below the toolbar. You can edit the logic as you would with most any text editor.
  • To check for correct syntax of your logic, simulate your model.

Logic Toolbar

The Logic toolbar provides additional functionality for modifying your logic. You must first select a line of logic, or multiple lines, in the logic window before using most of the buttons in the toolbar.

  • Statement List - this drop down list displays all valid statements for use in Logic for the selected object (see below).
  • Cut, Copy, Paste - the next three buttons in the toolbar allow you to cut (Ctrl+X), copy (Ctrl+C), and paste (Ctrl+V) lines of logic in the logic window.
  • Outdent and Indent - these two buttons allow you to align selected lines of logic, respectively. This feature can help in formatting the code to make it more structured and readable.
  • Comment and Uncomment - the next two buttons allow you to comment out and uncomment selected lines of logic. When you "comment out" a line of logic, it will not be executed. Process Simulator will ignore any lines of logic that have been commented out with double slashes (//).

Statement List

Every Logic window has a Statement List which shows you which statements can be used in the logic's context. Hovering over a statement in the list will open a help tooltip explaining what it does. Selecting a statement from the list will add it into the Logic window.

Automatic Formatting

Logic windows will attempt to automatically format your logic through indentation, coloring and capitalization. This is done for readability and consistency across models and users. For example, it formats the indentation of the logic within conditional statements such as the If, Else If, Else, and While-Do statements, by placing a starting brace { and ending brace } and indenting the statements between them. Currently, this default formatting cannot be customized.
