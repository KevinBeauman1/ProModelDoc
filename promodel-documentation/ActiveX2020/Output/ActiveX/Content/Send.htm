Send

Statement

Syntax samples

Send <expression> <entity name> To <destination>{,<priority>}

Send 2 EntA To Act2
Send 1 Grp_A To Grp_A_Processing, 10

Description

Sends the specified number of a particular entity type to the destination. The entities to be sent must be waiting with a Send routing. The entity that issued the Send command continues processing whether or not entities of the type requested are waiting to be sent. If no entities are waiting to be sent, a Send notice is automatically posted so that entities will be sent when they become available.

The Send statement can model a system based on demand, rather than on entity arrival, (called a pull system). Customer orders cause a Send to be issued for a main assembly. Main assembly issues Send commands for sub-assemblies.

The Send statement can also be used as a control device to limit the amount of work-in-progress (WIP) in certain critical areas. Quantities are only sent to the production area when the WIP level falls below a certain point.

If multiple Send statements are executed that trigger the sending of entities to the same activity, the order the entities are ultimately sent in could vary depending on the available capacity of that destination activity. When the destination activity has available capacity, the order in which the entities are sent would be the order the Send statements were executed.

Valid In

Any logic.

Components

<expression>

The number of entities to send to the destination. Negative values will generate an error message.

<entity name>

The type of entity to send to the destination. You may substitute Ent() for an entity name.

<destination>

The name of the activity to which the entities will be sent. You may substitute Loc() for the activity name.

<priority>

Multiple Send requests for the same entity type are filled according to the longest waiting request having the highest priority. This expression should be a number between 0 and 999.

Example

In this example, Loan’s arrive at InQueue and trigger the sending of DebtCheck entities to Research. The value of Loan’s attribute aNum determines the number of DebtCheck’s sent to Research.

Wait U(3, .5)
Send aNum DebtCheck To Research
