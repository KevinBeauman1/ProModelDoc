Arrival Properties

The Arrival properties define how entities enter the system at an activity. An arrival is defined by connecting an entity to an activity and setting the properties to define the arrival. If no arrivals exist in the process when the simulation starts, arrivals will automatically be created at activities with no incoming connection. However, as long as at least one arrival is defined, then other activities without arrivals defined will not have arrivals automatically created. The appearance of the Arrival properties will vary depending on the Type of arrival specified in the General section.

Right-click on a connector, choose Properties, and the Arrivals shape properties window displays.

There are four sections found in the Arrival properties window.

  • Calendar
  • Notes
  • Advanced

Properties is where you choose the kind of arrival you want to define. Select the type of arrival from the Type drop down list:

  • Periodic Arrivals occur at a specified time interval. This is the default arrival type.
  • Continuous An arrival occurs whenever the activity has available capacity.
  • Scheduled Arrivals occur at specified dates and times.
  • Ordered Arrivals only occur when an order is received for the entity triggered by a Storage activity.
  • Pattern Arrivals occur based upon a defined pattern of quantities over specified time segments.

Based on what type of arrival you select the details will vary.

Calendar

A calendar can be selected which will restrict the entities from arriving during the time off defined in the calendar. Calendars can only be selected for Periodic, Scheduled, and Pattern arrival types. The following options are available.

  • None No calendar will be assigned and the arrivals will occur based upon the Periodic, Scheduled or Pattern arrival definitions.
  • Specified at the arrival destination Selecting this option means that the arrival will follow the calendar assigned to the activity where the entities arrive.
  • Unique to this arrival By selecting this option, you specify which Calendar file the arrival will adhere to. The File field will appear after selecting this option where you can enter the Calendar path and file name or browse for and select the Calendar file.

Notes

Enter any notes you would like to make in regards to the arrival.

Advanced

Submodel Input This option is used in a submodel, and specifies that the arrival is an input of an entity from the parent model.
