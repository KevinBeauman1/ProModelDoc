Conveyance Kanban (Reorder Point) Control System

Cause entities to be held up until a signal is received that a downstream inventory has dropped to a specified inventory level.

 

The model used for this example, as well as all of the modeling "how-to" examples, can be found in the ProModel Solutions Café (http://www.promodel.com/solutionscafe/howto/).

Solution

  1. For the holding area which serves as the SOURCE of entity replenishment, define either an activity with an output buffer, an inventory, or a storage by dragging the appropriate shape onto the layout.
  2. For the buffer or storage that is to be replenished when its inventory drops to a specified trigger level, define a storage by dragging a storage onto the layout.
  3. Define a routing from the SOURCE activity to the storage, or to an intermediate activity that is one or more steps away from the storage.
  4. Specify the routing as an Ordered routing.
  5. Fill in the information for the storage.
Example

Large industrial pumps are drawn from a Pump Buffer and boxed. When the inventory of pumps in the buffer drops to 2, an additional 10 pumps are brought to the buffer from an adjacent Pump Inventory facility.

Solution

The Pump Buffer is defined as a storage. The routing from the Pump Inventory to the Pump Buffer is an Ordered routing. The storage dialog for the Pump Buffer has the appropriate trigger level and reorder quantity specified.
