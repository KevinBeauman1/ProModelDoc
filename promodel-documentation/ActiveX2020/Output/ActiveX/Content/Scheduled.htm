Scheduled

Scheduled arrivals allow you to define multiple periodic arrivals based upon a calendar date or weekly time.

Entity  Shows the entity connected to the arrival connector and specifies the type of entities that will arrive. If an entity is not connected to the arrival connection, then this field will be enabled and the arrival entity is manually selected. 

Click the Define Arrivals button to open the Scheduled Arrivals dialog.

First, choose the arrival basis, Weekly Time or Calendar Date, by which you will define your scheduled arrivals.

Next, click the Add button to add a scheduled arrival record to the grid.

Enter the Week number and week Day if the Scheduled Arrival basis is set to Weekly Time or, enter the specific Date if the basis set to Calendar Date. Entering the Time and Qty of the arrival completes the Scheduled arrival definition if only one arrival is desired.  Specifying the number of Occurrences and the Interval (i.e. frequency) between the arrivals is optional but when defined, allows you to repeat the arrival. If you use a scheduled arrival in your model, your simulation run length must be defined by either Weekly Time or Calendar Date, but not Time Only. See Simulation Options for more information.

Values for this grid can be copied and pasted to and from Microsoft Excel. To copy data from the grid you must first select the desired rows to copy.    Once the desired rows are selected, use the Ctrl+C hot key to copy the data to the Windows clipboard.  The data can now be pasted into Excel.  

When pasting data from Excel into the Scheduled Arrivals grid, use the Ctrl+V hot key to paste the data into the Scheduled Arrivals grid.
