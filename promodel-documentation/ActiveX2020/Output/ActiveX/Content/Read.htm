Read

Statement

Syntax samples

Read <file ID>, <element name>

Read File1, Var1

Description

Reads the next numeric value from a text file and assigns that value to an element name. Read files are defined in the External Files grid. When reading from a file, Process Simulator skips all non-numeric data, such as text, and reads the next numeric value. Thus comments and notes may be included in a read file. Multiple replications of a model will continue reading from a file where the previous replication left off unless reset with the Reset statement.

Please Note: Process Simulator reads a period (.) in a General read file as a zero. To avoid this, you should use the comment symbol (//) in front of notes and comments that contain a period.

Read can read text files. Most spreadsheet programs can convert spreadsheets to text files (.TXT) and comma-delimited files (.CSV).

If a read statement is not assigning the right values to the proper element, there may be numeric information in the read file’s header information (notes and comments). Additionally, if the values are being read into an array, the array indices may not be incremented properly between reads.

If a read file should be read more than once in a model, it may need to be reset. One way to tell when a file needs to be reset is with an end of file marker such as 9999 and the following two lines of logic.

Read File1, Value
If Value= 9999 Then Reset File1

Valid In

Any logic.

Components

<file ID>

The file identifier as defined in the External Files grid.

<element name>

The variable, array element, or attribute to be assigned the value.

Example

The example below shows a text file being read in Arrival logic. In this case three values representing the length, width and depth of each Box are listed on each line of a file called “Size.txt.” The File ID for this file is simply Size. Length, Width, and Depth are all attributes.

Read Size, Length
Read Size, Width
Read Size, Depth
