Activity Time

The activity can be defined by a simple activity time that is either a constant value or a probability distribution (normal, triangular, etc).

The model used for this example, as well as all of the modeling "how-to" examples, can be found in the ProModel Solutions Café (http://www.promodel.com/solutionscafe/howto/). A video demonstrating this solution can be found here.

Solution A (Using the Time Field)

  1. In the Time field of the activity’s Properties, type in the time value or right-click and select the Distribution Builder to specify a probability distribution and associated parameter values in the dialog that appears. The time unit (hour, minute, etc.) can be selected from the drop-down list box to the right of the Time field.

Solution B (Using Logic)

  1. Delete the default time that appears in the activity Time field. (This is important; otherwise, the entity will spend this time as well as any time defined in Logic.)
  2. Click on Logic at the top of the activity Properties window.
  3. From the statement list, select the Wait statement and type the time value.

Example

Orders go through a Review activity whose time is triangularly distributed with a minimum of .2 minutes, a mode of .5 minutes and a maximum of 1.8 minutes.

Solution A

The value T(0.2,0.5,1.8) is entered in the Time field of the activity Properties window.

Solution B

A Wait statement is defined in Logic with a time value of T(0.2, 0.5, 1.8).
