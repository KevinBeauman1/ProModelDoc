Location Activity Single Capacity State Pie Chart

Select LocationActivity Single Cap from the Pie Charts drop-down button within the State group in the Charts ribbon.

The following pie chart is generated.

This pie chart displays each state as a separate section of the pie based upon the percentage of overall simulation time the single capacity locationactivity spent in a state.

% In Operation: The percentage of time the locationactivity was actually processing an entity.

% Setup: The percentage of time a locationan activity was being set up during the simulation.

% Idle: The percentage of time the locationactivitywas available, but not in use.

% Waiting: The percentage of time the locationactivity was waiting for a resource, another entity or a wait until condition in order to begin processing or move to the next locationactivity. Any delays in processing move logic (even WAIT statements) are counted as waiting time, including Join, Load,attach actions and Match statements.

% Blocked: The percentage of time entities spent waiting for a freed destination.

% Down: The percentage of time thelocationactivity was not available due to unscheduled downtime.
