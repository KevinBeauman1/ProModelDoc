Location Activity Single Capacity States Table

Click on the Tables drop-down and select Single Cap States in the LocationActivity group.

A table is generated, which lists the percentage of time single capacity locationsactivities spent in various states during the simulation, and contains the following columns of information.

Scenario: The scenario to which the data in this row applies.

Name: The locationactivity name.

Scheduled Time: The total amount of time the locationactivity was scheduled to be available. This value is now in decimal format, not truncated. (Excludes off-shift time, break time, and scheduled downtimes.)

% Operation: The percentage of time the locationactivity was processing an entity.

% Setup: The percentage of time the locationactivity spent in setup in order to process the entities.

% Idle: The percentage of time no entities were at the locationactivity, but the locationactivity was not down.

% Waiting: The percentage of time the locationactivity was waiting for a resource, another entity, or a WAIT UNTIL condition in order to begin processing or move to the next locationactivity. Any delays in processing move logic (even WAIT statements) are counted as waiting and include JOIN, LOAD,attach actions and MATCH statements.

% Blocked: The percentage of time entities spent waiting for a freed destination.

% Down: The percentage of time the locationactivity was down as the result of unscheduled downtimes. This does not exclude the possibility of overlap with any of the previous three states.
